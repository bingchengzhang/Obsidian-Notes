### 运行程序的流程
**1. 预处理 (Preprocessing)**

- **做什么：** 编译器在正式看代码之前，先做一些文本替换工作。比如把 `#include` 的头文件内容直接复制进来，展开所有的宏定义（`#define`），并删除代码里的注释。
    
- **产出：** 一个纯净的、完整的源代码文件。
    

**2. 编译 (Compilation) —— （变量声明在这里被处理）**

- **做什么：** 这是最复杂的一步。编译器会进行词法分析、语法分析、语义分析。它会检查你有没有写错别字、语法对不对。**同时，编译器会建立“符号表”，记录所有声明的变量及其类型**，并最终将高级语言翻译成低级的**汇编语言（Assembly）**。
    

**3. 汇编 (Assembly)**

- **做什么：** 汇编器将人类还能勉强看懂的汇编语言，翻译成计算机 CPU 唯一能懂的 0 和 1（机器指令）。
    
- **产出：** 目标文件（Object File，例如 `.o` 或 `.obj` 文件）。
    

**4. 链接 (Linking)**

- **做什么：** 你的程序通常不会只用自己写的代码，还会调用标准库（比如 `printf` 或 `console.log` 的底层实现）。链接器负责把你的目标文件和系统库的其他目标文件“缝合”在一起。
    
- **产出：** 最终的可执行文件（如 Windows 的 `.exe` 或 Linux 的 ELF 文件）。

**5. 装载与执行 (Loading & Execution) —— （时间复杂度在这里体现）**

- **装载：** 操作系统读取可执行文件，将其加载到内存（RAM）中。操作系统会为你划分好代码区、数据区，并初始化程序的“调用栈（Stack）”和“堆（Heap）”。
    
- **执行：** CPU 的程序计数器（PC）指向程序的第一条指令（通常是 `main` 函数的起点）。CPU 开始进入“取指 - 译码 - 执行”的无限循环，直到程序结束。**我们在算“时间复杂度”时，评估的就是 CPU 在这一步中，核心代码循环执行指令的次数趋势。
